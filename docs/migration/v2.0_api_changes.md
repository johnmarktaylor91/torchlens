# TorchLens 2.0 API changes

TorchLens 2.0 reserves exactly 40 top-level names in `torchlens.__all__`. Names
outside that budget either moved to a namespace with a deprecation shim, remain
available only from their canonical namespace, or are new reserved stubs for a
later feature-overhaul phase.

## Wrapper migrations

## Phase 1b: option classes + canonical vocabulary

Grouped options are now available under `torchlens.options`. Individual kwargs
continue to work for the migration window, but option objects are the preferred
style for new code.

| Entry point | Old kwarg | New option-class field / canonical kwarg |
|---|---|---|
| `log_forward_pass` | `layers_to_save` | `capture=CaptureOptions(layers_to_save=...)` |
| `log_forward_pass` | `keep_unsaved_layers` | `capture=CaptureOptions(keep_unsaved_layers=...)` |
| `log_forward_pass` | `output_device` | `capture=CaptureOptions(output_device=...)` |
| `log_forward_pass` | `save_function_args` | `capture=CaptureOptions(save_function_args=...)` |
| `log_forward_pass` | `save_gradients`, `gradients_to_save` | `capture=CaptureOptions(save_gradients=..., gradients_to_save=...)` |
| `log_forward_pass` | `random_seed`, `detect_loops` | `capture=CaptureOptions(random_seed=..., detect_recurrent_patterns=...)` |
| `log_forward_pass` | `mark_input_output_distances` | `capture=CaptureOptions(compute_input_output_distances=...)` |
| `log_forward_pass` | `activation_postfunc` | `save=SaveOptions(activation_transform=...)` |
| `log_forward_pass` | `gradient_postfunc` | `save=SaveOptions(gradient_postfunc=...)` |
| `log_forward_pass` | `save_raw_activation`, `save_raw_gradient` | `save=SaveOptions(save_raw_activation=..., save_raw_gradient=...)` |
| `log_forward_pass` | `save_activations_to`, `activation_sink` | `streaming=StreamingOptions(bundle_path=..., activation_callback=...)` |
| `show_model_graph` / `show_backward_graph` | `vis_mode` | `view` or `VisualizationOptions(view=...)` |
| `show_model_graph` / `show_backward_graph` | `vis_nesting_depth` | `depth` or `VisualizationOptions(depth=...)` |
| `show_model_graph` / `show_backward_graph` | `vis_renderer` | `renderer` or `VisualizationOptions(renderer=...)` |
| `show_model_graph` / `show_backward_graph` | `vis_node_placement` | `layout` or `VisualizationOptions(layout=...)` |
| `show_model_graph` / `show_backward_graph` | `vis_node_mode` | `node_style` or `VisualizationOptions(node_style=...)` |
| `ModelLog.replay`, `ModelLog.replay_from`, `ModelLog.rerun` | `strict`, `hooks`, `append` | `replay=ReplayOptions(strict=..., hooks=..., append=...)` |
| `ModelLog.do` | `engine`, `confirm_mutation`, `strict` | `intervention=InterventionOptions(engine=..., confirm_mutation=..., strict=...)` |

Before:

```python
log = tl.log_forward_pass(
    model,
    x,
    layers_to_save=["fc1"],
    activation_postfunc=lambda t: t.detach().cpu(),
    vis_mode="rolled",
    vis_nesting_depth=2,
)
```

After:

```python
log = tl.log_forward_pass(
    model,
    x,
    capture=tl.options.CaptureOptions(layers_to_save=["fc1"]),
    save=tl.options.SaveOptions(activation_transform=lambda t: t.detach().cpu()),
    visualization=tl.options.VisualizationOptions(view="rolled", depth=2),
)
```

Forward validation:

```python
# Before
import torchlens as tl
ok = tl.validate_forward_pass(model, x, None, 42)

# After
from torchlens.validation import validate_forward_pass
ok = validate_forward_pass(model, x, None, 42)
```

Backward validation:

```python
# Before
ok = tl.validate_backward_pass(model, x, None, loss_fn)

# After
from torchlens.validation import validate_backward_pass
ok = validate_backward_pass(model, x, None, loss_fn)
```

```python

```
